# WCAG 2.1 AA Accessibility Improvements

This PR adds comprehensive accessibility improvements to the OnboardingFlow component.

## Changes

### Semantic HTML
- Added skip-to-content link for keyboard navigation
- Added proper ARIA landmarks (`main`, `section`, `region`)
- Converted bullet lists to semantic `<ul>` and `<ol>` elements
- Added proper heading hierarchy with `aria-labelledby`

### Focus Management
- Added visible focus indicators (`focus:ring-2 focus:ring-offset-2`)
- All interactive elements now have clear focus states
- Improved keyboard navigation experience

### Form Accessibility
- Added `aria-required` to required fields
- Added `aria-invalid` for error states
- Added `aria-describedby` linking inputs to error messages
- Improved error messaging with `role="alert"` and `aria-live="assertive"`

### Progress Indicators
- Progress bar now uses `role="progressbar"` with proper ARIA attributes
- Added `aria-valuenow`, `aria-valuemin`, `aria-valuemax`
- Step list uses `aria-current="step"` for current step
- Progress percentage announced to screen readers

### Screen Reader Improvements
- Non-critical info boxes use `role="status"` with `aria-live="polite"`
- Critical errors use `role="alert"` with `aria-live="assertive"`
- Success state clearly announced
- All steps and progress are properly labeled

### Color & Contrast
- Improved text contrast for better readability
- Better visual hierarchy with proper spacing
