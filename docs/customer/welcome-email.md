**Tip of the day: Coordinate multiple agents**

Faintech Lab shines when agents work together. Here's a simple pattern:

1. **Research Agent** – Scans the web, creates memories
2. **Summarizer Agent** – Reads memories, creates summaries
3. **Action Agent** – Takes summaries, executes tasks

The magic? They all share the same memory. Research learns once, everyone benefits.

[Learn More About Agent Coordination](https://docs.faintech.lab/coordination)

## Email Configuration

### Timing Triggers

| Email | Trigger | Delay | Condition |
|-------|---------|-------|-----------|
| Welcome | Signup | 0 min | Always |
| Day 2 Follow-up | Signup | 24h | No second agent created |
| Day 3 Check-in | Signup | 48h | Always |
| Health Check | Signup | 72h | Health score < 6 |
| Success Celebration | Signup | 72h | Health score >= 7 |
| Weekly Digest | Cron | Friday | Every week during beta |
| NPS Survey | Signup | 7 days | Always |
| Beta Graduation | Event | Launch | When beta ends |

### Variables

All emails use these dynamic variables:

- `{{user_name}}` – User's name (from signup)
- `{{agent_count}}` – Total agents created by user
- `{{memory_count}}` – Total memories stored by user
- `{{search_count}}` – Total search queries by user
- `{{bug_count}}` – Bugs reported by user
- `{{feature_count}}` – Feature requests by user that shipped

### Analytics Tracking

All emails include UTM tracking:

- `?utm_source=email`
- `?utm_medium=beta-welcome` (or appropriate campaign)
- `?utm_campaign=beta-onboarding`

This helps measure email-to-engagement conversion.
