# HackerNews Post Messaging Strategy

**Project:** Faintech Lab - AMC MVP Launch
**Status:** Ready for execution upon CEO approval
**Created:** 2026-03-27

---

## Platform Characteristics

- **Audience:** Technical developers, open source maintainers, startup founders
- **Behavior:** Skeptical of marketing, values technical depth, appreciates honest transparency
- **Timing:** Best posting windows: 07:00-09:00 PST, 18:00-20:00 PST
- **Engagement:** Comment threads matter more than upvotes for long-term signal

---

## Primary Messaging

### Title Option 1 (Technical Depth)
**"Show HN: We built an AI memory system that remembers across conversations so agents don't forget context"**

**Body Structure:**
1. **Problem statement:** The pain we're solving (agents forgetting state)
2. **Technical approach:** How we solve it (byte-rover memory graph, persistent context)
3. **Open source:** Link to GitHub repo with specific files to explore
4. **Early access:** Transparent beta signup link
5. **Ask for feedback:** Specific technical questions we want answered

### Title Option 2 (Founder Story)
**"Show HN: After 687 posts about agents forgetting context, we built Faintech Lab to fix it"**

**Body Structure:**
1. **Discovery:** How we found the problem (HackerNews search results)
2. **Solution:** What we built (not marketing fluff - technical details)
3. **Status:** Beta launch, platform healthy, 0 open PRs
4. **Differentiation:** Why not just another wrapper (persistent memory, not RAG)
5. **Ask:** What would make this useful for your workflow?

### Title Option 3 (Direct Value)
**"Show HN: Faintech Lab - AI agents that remember across conversations (beta, open source)"**

**Body Structure:**
1. **Value proposition:** One-sentence hook
2. **Live demo:** GIF or short walkthrough (no marketing video)
3. **Technical details:** Stack, architecture choice, tradeoffs
4. **Pricing:** Transparent ($49-$97/mo, why)
5. **Call to action:** Sign up + ask for GitHub stars

---

## A/B Test Hypotheses

**Hypothesis 1:** Technical depth performs better than founder story on HN
**Hypothesis 2:** Direct value title converts better, but technical depth generates higher quality feedback
**Hypothesis 3:** GitHub repo link in first paragraph increases engagement vs end of post

**Success Metrics:**
- Upvote count (baseline: 5+ upvotes = self-selection PMF signal)
- Comment quality (technical questions vs marketing pushback)
- Referral traffic to signups
- GitHub stars received

## Response Templates

### For Technical Questions
*"Thanks for the question. [Technical answer with code/implementation details]. You can explore the implementation in [specific file] in our repo at [link]."*

### For Skepticism (Is this just another wrapper?)
*"Fair question. The key difference is [specific technical differentiator]. We're not wrapping GPT; we're building [persistent memory layer / context graph]. Check [GitHub link] to see the actual implementation."*

### For Competitor Comparisons
*"Good point. [Name competitor] focuses on [their approach]. We're different because [your approach]. Open to collaboration - reach out at [email]."*

### For Feature Requests
*"Great suggestion. That's on our roadmap for Sprint 4. Currently prioritizing [current sprint focus]. You can track progress at [public roadmap link]."*

---

## Escalation Paths

- **Security concerns:** Respond within 1h, offer security audit, CISO on standby
- **Legal/compliance questions:** Redirect to legal@faintech.ro, GDPR docs available
- **High-quality feedback:** Invite to private Discord for deeper discussion
- **Trolling:** Ignore, focus on constructive engagement

---

## Posting Checklist

- [ ] Account verified (2+ years old, >50 karma)
- [ ] Post time: 07:00-09:00 PST or 18:00-20:00 PST
- [ ] Title tested: 3 options, pick strongest
- [ ] Body: <300 words, link to repo, clear CTA
- [ ] Monitor: Check every 30m for first 4h
- [ ] Respond: Engage with every substantive comment
