# new-product

New customer-facing product. Direction set by CPO based on market research.

## Status: Discovery

CPO owns this. Current focus: identify the right problem to solve.

## Constraints
- Must be distinct from FainTrading (no trading/finance overlap)
- Should leverage what we learn from meta-ai (AI-native product)
- Target: technical teams or founders who can self-serve
- Revenue model: SaaS or usage-based

## Process
1. CPO + Market Research: define 2-3 candidate problems with evidence
2. CEO: pick one and set a 4-week MVP bet
3. Dev + Frontend + Backend: build the smallest thing that tests the hypothesis
4. Growth: get 10 real users, measure, decide

## Output format
Every experiment gets a file in `docs/experiments/YYYY-MM-DD-<hypothesis>.md` with:
- Hypothesis
- What we built
- What happened
- Decision: continue / pivot / kill
